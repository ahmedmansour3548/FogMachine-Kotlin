FogMachine

The Fog Machine is a simple, black box library designed to generate GeoJSON files that comprise of square Polygons or MultiPolygons with square holes. It offers extensive customization by leveraging Kotlin's DSL functionality. Below are a few examples of how to use this program, and their expected results. The generated GeoJSON file and reference image can be found here. 

              
zone {
  path = "zone"
  longitude = "-73.98"
  latitude = 40.77
  tyleCoarseness = Coarseness.MEDIUM
  zoneCoarseness = Coarseness.COARSE
  holes {
      coordinates = listOf(Pair("-73.974", "40.776"), Pair("-73.973", "40.776"))
      ids = 5
  }
  properties {
      property {
          name = "prop1"
          value = 0
      }
      property {
          name = "prop2"
          value = JSONObject("""
              {
                  "city": "New York",
                  "country": USA,
                  "park": "Central Park"
              }
              """.trimIndent())
      }
  }
}

Result: Creates a Zone GeoJSON file in the current directory with name "zone.geojson". Its bottom left corner is at longitude -73.98, latitude 40.77. It includes 3 holes, one at Tyle ID: 5. and two adjacent ones at longitude -73.974, latitude 40.776 and longitude -73.973, latitude 40.776. It also inserts properties into each Tyle Polygon, one being a Number and another a JSONObject. Note that the longitude and latitude can be any numerical represented object, such as Floats, Doubles, or Strings. 
            
myst {
  path = "myst.geojson"
  zoneCoarseness = Coarseness.MEDIUM
  holes {
      coordinates = Pair("-73.973", "40.773")
      ids = 28037605974
  }
  properties {
      property {
          name = "prop1"
          value = "value1"
      }
      property {
          name = "prop2"
          value = null
      }
  }
}

Result: Creates a Myst that covers the Earth except for two adjacent Zones at longitude -73.973, latitude 40.773. The Zone ID 28037605974 was generated using a modified Cantor Pairing Function. 
            
zone {
  path = "zone2.geojson"
  coord lng -73.97  lat "40.77"
  coarseness tyle FINE zone MEDIUM
  holes {
      coordinates = "-73.9692, 40.7705"
      ids = listOf(85, "86", 95, 96)
  }
  properties {
      property {
          name = "prop1"
          value = JSONArray("[23, 55, 13]")
      }
      property {
          name = "prop2"
          value = true
      }
  }
}

Result: Creates a Zone with its bottom left corner at longitude -73.97, latitude 40.77, and with numerous holes. Note the infix functions used to define the starting coordinate as well as the coarseness. Also note the flexibility of allowed data types for numerical values, and the ability to set property values as JSONArrays and boolean values. It is recommended to use BigDecimal or String representations for numerical values as often as possible to reduce unintended behavior due to rounding errors.
